三層知識解耦架構

每個工具只負責一件事，互不重疊——這是幻覺率大幅降低的根本原因。

🗄️
Obsidian
儲存層

結構化知識 + 雙向連結 + BM25 路徑召回。只管「資料在哪、長什麼格式」，磁碟 I/O ↓99%。

🧠
NotebookLM
理解層

Source-grounded 語意問答，段落級來源綁定。只管「語意是什麼、引用哪篇」，引用完整性 67% → 100%。

⚡
Claude Code
執行層

AI 代理 + 自動化腳本 + 跨 Session 記憶。只管「現在做什麼、如何操作」，Token ↓76%。

五層防幻覺框架

覆蓋知識全生命週期的系統性防禦鏈，幻覺率 59% → 3%（-95%）。

1
攝取時 — Source-grounded 來源綁定 NotebookLM 強制每個答案附段落級來源標注，封鎖輸入端幻覺
2
檢索時 — BM25 詞彙精確召回 bm25_search.py 路徑召回，避免全文注入，查詢 0.38–0.41 秒
3
寫入時 — CLAUDE.md 行為約束 每次開工自動載入規範，確保 AI 代理遵守引用規則
4
審計時 — hallucination_guard.py 全文掃描 偵測無來源引用的百分比數據與研究聲明，112 警告 → 0
5
一致性 — consistency_check.py 跨章節稽核 自動偵測術語矛盾，確保整篇論文用語統一

七支自動化腳本

安裝完成後即可使用，全部在 scripts/ 資料夾。

perpetual-rag init

建立新知識庫，自動生成所有系統文件

perpetual-rag search

BM25 精準關鍵字搜尋，0.4 秒回傳結果

perpetual-rag audit

防幻覺掃描，標記無來源引用的論點

perpetual-rag graph

生成知識圖譜，輸出 Canvas / JSON / DOT

perpetual-rag bump

互動式版本升級，自動更新日誌與責任記錄

consistency_check.py

跨章節術語一致性稽核，自動偵測矛盾

daily_audit.py

每日進度快照，知識庫健康狀態監控

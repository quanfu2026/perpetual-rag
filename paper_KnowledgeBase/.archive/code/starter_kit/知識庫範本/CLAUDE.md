# CLAUDE.md — AI Agent 行為準則

> 修改此檔案來定義你的 AI 代理行為邊界。
> 每次 Claude Code 啟動都會讀取這份規範。

---

## 一、授權目錄

```
工作目錄：~/my_KnowledgeBase/    ← 修改為你的實際路徑
```

所有讀取、分析與編寫操作，嚴格限制在此目錄之內。

---

## 二、回寫與同步規則

- **完成任務後**：同步更新 `sessionhandoff.md` 和 `index.md`
- **新建筆記時**：加入 YAML frontmatter（title / type / status / tags）
- **收工時**：將進度完整寫入 `sessionhandoff.md`，確保下次開工可以繼續

---

## 三、命名規範

| 類型 | 格式 | 說明 |
|------|------|------|
| 參考文獻 | `REF_作者年份_關鍵詞.md` | 自動分類為參考文獻節點 |
| 章節筆記 | `第X章_主題/節號_標題.md` | 自動分類為章節節點 |
| 系統報告 | `00_報告名稱.md` | 自動排除於知識圖譜之外 |

---

## 四、禁止行為

- **禁止產生孤立文件**：所有新文件必須在 `index.md` 中登記
- **禁止刪除決策日誌**：`index.md` 歷史記錄不可刪除
- **禁止在未讀取 sessionhandoff.md 前開始工作**：每次開工必須先讀取記憶交接檔

---

## 五、任務分派等級（L1–L4）

| 等級 | Token 預算 | 適用任務 |
|------|-----------|---------|
| L1 | 1–3K | 快速查詢、單檔修改 |
| L2 | 3–10K | 章節撰寫、腳本執行 |
| L3 | 5–15K | 跨章節整合、多檔操作 |
| L4 | 10–30K | 全稿稽核、系統重構 |

超過 L4 範圍的任務請拆分後分次執行。

---

## 六、五層防幻覺規範

1. **攝取時**：上傳到 NotebookLM 的文件必須是原始來源（PDF 或官方文件），不可上傳二手整理
2. **撰寫時**：每個百分比數據、研究聲明必須附上 `[[REF_來源]]` 雙向連結
3. **完成後**：執行 `python3 scripts/hallucination_guard.py` 掃描警告
4. **跨章節**：執行 `python3 scripts/consistency_check.py` 確認術語統一
5. **收工前**：零警告才可標記章節為 `status: final`
